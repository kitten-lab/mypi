# Shell / narrative locks

## Phrase (check gate)

When hands says something like:

> **hard lock the narrative and vibe of this shell — use it moving forward for expansions**

…that shell is **locked**. Agent expansions (new rooms, tools installed on that SYS, chrome tweaks) must **match the locked voice and metaphors**. Do not re-skin, re-theme, or “improve” the prose to a different register.

Unlock only if hands explicitly says so (or rewrites the lock entry below).

## What “locked” means

| Do | Don’t |
|----|--------|
| Read shell + room copy before expanding | Overwrite ledes with underwire (`parent_c_uid`, getTool notes) |
| Match tone, metaphors, punctuation density | Import vibe from another SYS (book leather ≠ terminal) |
| Small bugfix that preserve voice | “Helpful” rewrite of narrative copy unasked |
| New rooms in the same dialect | Status-board / DATA·DATA·DATA copy when the room doesn’t need a lede |

## Locks

### terminal — LOCKED

- **Canon line:** *the forest remembers* (status bar; keep / echo that register).
- **Also in shell:** *for the collection and protection of your thoughts* · *put them here* · *who are you*
- **POV:** a strange girl who found a terminal and logged in — not a sysadmin dashboard, not a product tour.
- **Feel:** otherworldly, slightly unsettling, intimate. Silence is allowed. The machine is not helpfully directive.
- **Rooms:** prefer **title + tool only**. Do not invent `operator X. file store. write. commit.` style ledes. If a line exists, it should feel like the machine speaking (or withholding), not a manual.
- **Avoid:** desk/paper cosiness; DATA · DATA · DATA filler; explaining how the tool works; “you should now…”.
- **Paths:** `a/terminal/**`, `m/doors/terminal/**`
- **Agent note:** hands may still rework shell chrome; when expanding, re-read shell first and match *that* voice, not a prior DATA register.

## Hands rework vs agent

Hands reworks shells and room copy freely. Agent does not need constant “hands off” — only the **hard lock** freezes narrative dialect. Unlocked shells: re-read before editing copy; match disk.
