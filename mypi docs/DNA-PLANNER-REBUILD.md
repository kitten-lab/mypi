## 2. DNA pillars (the pattern that was supposed to hold)

### 2.1 Claim → shell (asSys / asDom)

**Law (recover):**

> A page can live **anywhere** and **claim** a place (historically `sys` / surface).  
> That claim **pulls the facility shell** (`asSys`).  
> A place may add its own look (`asDom` stylesheet) — **favorite colors / identity**, not a baked lookbook that forbids fonts and padding.

| Piece | Job |
|-------|-----|
| **asSys** | Facility shell for that surface (structure, mast, slots) |
| **asDom** | Optional place-skin on top of the shell (identity, color, local marks) |
| **Claim** | “I am of this surface” → load shell; not “I am a one-off website” |

**Corruption to refuse:** stuffing every skin into `terminal/asSys/` as permanent default AI chrome; skins that cannot break padding/type; pages that never claim.

**Reference recovery targets:** early WWW/DEMO claim patterns; silo archives; `WWW-RESTORE.md`; interra/www `asSys` restore notes; any surviving `asDom` sheets outside a single bunker.

### 2.2 Tool = jack (composability)

**Law (recover):**

> A **tool** is a **portable function pack** (getTool name + Fn), not a full product website.  
> Same tool, different surfaces, different reasons.  
> Facility owns **chrome** (brand bar, rail labels). Tool owns **slots** (Rail controls, Desk body).

| Piece | Job |
|-------|-----|
| **getTool(tool, Fn)** | Load page/actor/script for that Fn only |
| **Rail / Desk / …** | Named Fns, not one mega-page that owns the house |
| **Shell** | Places the tool into the room |

**Corruption to refuse:** tool PHP that hardcodes facility slogans; dual-pane SaaS as default “good design”; tools that cannot move.

**Reference recovery targets:** `chesters-toy-box` carl-core, data-forester; pure getTool-era tools; mailroom only *after* shell/tool split is honest.

### 2.3 Alphabet / place network

**What it was for:** living grammar of **where** and **who speaks** — walkable identity internet — not folder cosplay.

**Retired domination words (sealed intent):** `sys` / `dom` / `mod` as *product speech* for hierarchy of control — see [PLACE-AND-NAMING.md](./PLACE-AND-NAMING.md). Prefer **surface / destination**, **place_path**, **room**, **voice/mask**.

**Tension (document, don’t fake-resolve):**  
Historical magic was “claim `sys=SKYLINE`.” New speech wants non-domination words. DNA must say:

> **Claim still exists.** The *string* of the claim may migrate (`surface=SKYLINE` or path root) but **pull-shell behavior must remain**. Renaming without the claim ability is just vocabulary cosplay.

### 2.4 Surface = walkable memory archive

**Law:** the surface is where you **store and walk** materials — not only “apps that post to a DB.”

Terminals, doors, ROMs, yards: **rooms of memory and work**, not a random SPA farm.

## 7. Greatest concern — DB must not become next glass

If the ledger is only “whatever got written under wrong names and wrong tools,” the next rebuild inherits **another testament of cope**.

### Protocol (DNA-level)

| Rule | Meaning |
|------|---------|
| **1. Label the era** | Current ledger = **ERA_PI_CONTAMINATED** (or your name). Not “production forever.” |
| **2. Dual speech** | Docs say CHESTER_UID / leaf-branch-log; code may still say `c_uid` until a deliberate migrate. |
| **3. Export faces, not only blobs** | Anything precious gets **face_id / human code / scale kind** on the way out — so next house isn’t opaque hex soup. |
| **4. No silent growth of wrong kinds** | New writes should set **kind + scale** consciously; refuse mystery rows as “default crate.” |
| **5. Glass stays glass** | Chat exports remain LOG-scale glass; ledger is not a dump of chat. IMPORT is a **decision**, not automatic bleed. |
| **6. Rebuild may re-import** | Next surface may re-ingest with **your** names; contaminated era is a **source**, not the identity of the next DB. |

**Success:** you can leave the current DB behind without feeling you abandoned yourself — because DNA + exports carry the *meaning*, and the SQLite file is **an era**, not the soul.
