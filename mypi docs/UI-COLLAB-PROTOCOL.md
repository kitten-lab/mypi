# UI collab — reach back, don’t replace

When you point at an old surface (forester, carl-core, …):

> **“I already chose things there I liked. Study those files. Don’t waste that effort.”**

That is **not** “apply a vibe” or “what good design means.”

## Rules

1. **Open the reference** (HTML + CSS). Port **bones** first: where brand vs nav vs rail vs rows live, font size, button scale, border weight.
2. **Color is last** (or yours to prove any palette). Recolor alone is not the ask.
3. **Anti-pattern sentence** if needed: e.g. *not list|detail SaaS*.
4. **No slogan pills** in brand-bar slots meant for Login/O/X.
5. **Small on purpose is intentional** — don’t “improve” to 14px product type.
6. If unsure: paste a screenshot or path + “copy the hand of this,” not “make it nice.”

## Brand bar vs nav (mailROOM example)

| Zone | Holds |
|------|--------|
| Brand bar | House mark + **session** (Login / Logout / O / X) |
| Side rail **chrome** | Facility labels (MAIL/ROOM, CHAR/LIE) — **shell owns** |
| Side rail **tool** | Queues + filters — **getTool Rail** |
| Yard + manage | Rows + object panel — **getTool Desk** |

Nav does **not** live in the brand strip.  
Facility slogans do **not** live in tool PHP.
