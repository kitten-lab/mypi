## 1. Product split (claims)

### 1.1 Three faces of the ecosystem

From face 600 (mockup walkthrough):

| Surface | Audience | Job |
|---------|----------|-----|
| **AIDM Studio** | creators | Creation **operating system** / engine room |
| **AIDM Play** | players | Inhabit world — no VENEdit, no sim debugger |
| **AIDM Social** | community | Profiles, lookbooks, discovery (MySpace-like; **separate** from Studio) |

> “Separate — like this is the studio, and the game-app is different, and the social network… can be played from mobile… or desktop app.” — face 600 user

Studio is **not** the game. Studio is not the social network.

### 1.2 Studio as creator OS

Tools named in the mockup stack (face 600):

| Tool | Role |
|------|------|
| **VENEdit** | Entity editor |
| **AIDM Chat** | AI copilot / reasoning (modes later) |
| **SIM Play** | Runtime simulation |
| **EVE Graph** | World analytics |
| (later) | Property creator, watcher grids, timeline inspector, pack editors |

Creator loop in mockup:

```
[VENEdit]  ←→  [AIDM Chat]  ←→  [SIM Play]
         aidm run>sim:…
```

### 1.3 Spatial workspace (canvas)

User-driven UX claims (face 600):

- Zoomable / pannable **grid canvas** of apps (not only tabs)
- Zoom into an app full-scale **without losing** its place on the big grid
- Connector line / breadcrumb from zoomed view back to canvas location
- Taskbar chips = running apps; click chip → jump to app location
- Spatial zones (entity editing / simulation / analytics) as studio wall

Comparisons used in glass (Figma/Miro-like) — **apps on canvas**, not only sticky notes.

### 1.4 Social is offline from Studio

Pipeline sketched in glass:

```
AIDM Studio → world simulation → vens → encounters/memories → lookbooks → profiles/social
```

Social content is **generated by play** (moments, lookbooks), not manufactured posts. Profiles = adventure archives. NPC lookbooks can develop followings.

**Hosting split (face 600 discussion):** `imported.to` / world paths as creator-controlled vs player content under guild/timeline — open product policy, not signed here.

## 2. What this is *not* (boundary)

- Not a request to build full multiplayer GPU client now  
- Not a mega-Desk that owns mailroom / mythleak / SOPER  
- Not auto-headlines from chat titles  
- Not “AIDM = current chat persona” product identity — Studio is **tools + surfaces**
