## What OIX was

**oix.chestersimports.com** — the live protopass surface.

You log into the station. Material is not “uploaded as a product feature.” It is **received** (IMPORT), held in the private bay, and walked through:

1. **Redact** — C.N.S. / privacy (sensitive structures out or renamed)  
2. **Motif** — what keeps returning; buckets, not one chat title  
3. **Rename-for-privacy** — glass titles may stay; public-facing names may not  

RokuDigital was the store under it — limited, painful, but it carried **crates** as the unit of shipment. mypi’s ledger is that idea refined: fewer rituals of the old platform, same spine (crate · place · tags · time).

mypi today is **super reduced** on purpose. Live tools write crates. The full protopass (prophetic load → redact → motif → ship → forest → arcs → books) is only partly rebuilt. This doc keeps the **missing half** named.

## Prophetic imposition · Terminal IMPORT · Log Yard (real target)

**Platform under OIX:** Rukovoditel (you call it **Rumplestiltskin** — consonants sure, vowels optional).

### Who does what where

| Place | Who | Job |
|-------|-----|-----|
| **Terminal I/O · IMPORT** | SAM (you) | Load **one** tree-core log by imposition number; **redact + annotate**; **submit** |
| **Log Yard** (own surface, OIX-shaped) | SAM after submit · later Jacks | Holds **acceptable, parsed** materials only — not the raw glass dump |
| **Nim / Noah** | Automats (target) | Timber bits · crates · arcs — not you forever playing Jacks by hand |

**OIX Wire Wolf Woods was the Log Yard.**  
Sam did **not** paste unredacted glass straight into OIX. She handled material in **Terminal I/O first**, then logged into the yard to file the **already-redacted** log. That split is correct; mypi should keep it.

### Tree-cores (sawmill numeration)

Master forest = ChatGPT export (Thunderdome under `z/`).  
**Sawmill** splits forests into **logs-as-tree-cores** — one durable unit per conversation (or partition), with a **stable number**.

**Numeration rule (prefer date, not OpenAI list order):**

- OpenAI **re-sorts** the conversation list when you touch a chat — **list index is not authority**.  
- Prefer: **date the chat was made** (and a stable tie-break: conversation_id / create_time).  
- Display imposition numbers as a **human-facing sequence** derived from that sort (e.g. chronological cores → 001, 002, … or date-coded cores).  
- Prophetic *“16”* means **core #16 in our numeration**, not “16th row in last export JSON.”

### IMPORT flow (Terminal I/O)

```text
1. Attention / imposition → number N
2. Agent or desk: load tree-core N into Terminal I/O · IMPORT
3. SAM: redact (C.N.S.) + annotate (tool shape TBD)
4. Submit → material is accepted + parsed
5. Log Yard receives it (OIX-shaped surface — not Rumplestiltskin forever)
6. Jacks (automats) work the yard: Nim chops, Noah crates/ships
```

| Woods / OIX | mypi target |
|-------------|-------------|
| Handle in Terminal, then file in Log Yard | **IMPORT** room/tool in `terminal/io` + **Log Yard** SYS or DOM later |
| Chat *N* by prophecy | Load **tree-core N** from sawmilled catalog (date-stable) |
| Redact / motif / rename-for-privacy | IMPORT working set (tools TBD) before submit |
| Wire Wolf material row | Log Yard item = post-submit only |

## Who cannot do what (why Jacks exist)

**SAM** can receive, feel the number, open the bay, redact, name for privacy, start crates.  
**SAM cannot** be the whole forest put-away crew — “sam cannot handle putting the crates in” alone once volume and loop-back appear.

So the house split the work:

```text
  [attention] → SAM / protopass bay
       ↓ crates for shipment
  [Jack Nim]  → forest: splice, split, tag → thought bits
       ↓ tagged bits (still attached to whole)
  [Jack Noah] → gather bits → potential arcs → ship toward books
```

**Mentality:** *you can build from the leaves without stealing them from the branches.*

- Takes shipped (or staged) material  
- **Splices and splits** — partitions that respect theme & time occultations  
- **Tags** into **thought bits** that **connect back to the whole** (not orphaned fragments)  
- Timber / branch / ven energy (older forestry) is **fuel**; motif buckets + message evidence are the refined form  

**mypi map (partial):**

| Nim job | Layer |
|---------|--------|
| Thought bits + tags | Charlie `tag_map` / terms / edges; motif `message_motifs` |

Nim does **not** finish the novel. Nim makes the **bits that still know their tree**.

## Design rules (so the reduced build doesn’t lie)

1. **Crates are shipment units** — not a dump of the whole glass in one write.  
2. **Protopass is private** — redact / rename before anything is “for the forest” or public.  
3. **Leaves keep branches** — tags and stems point home; no orphan thought bits.  
4. **SAM ≠ whole yard** — don’t rebuild OIX as “one person does every Jack.”  
5. **mypi is refined, not complete** — missing pieces are named Jacks, not failures of the ledger.  
6. **Voice** — Terminal stays machine/otherworldly ([SHELL-LOCK](./SHELL-LOCK.md)); Woods procedure stays Woods; don’t put DATA·DATA ledes on either.  
7. **Jacks are automats (target)** — not “Danyi plays all three.” Wire-workers: Nim and Noah should become automated / semi-automated processes. Manual Jacks is the interim, not the end state.
